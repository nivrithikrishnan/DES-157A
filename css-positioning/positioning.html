Understanding Relative and Absolute Positioning in CSS

Example 1

Below there is a parent element that is set to relative positioning. Inside that element is another element set to absolute postioning.

Absolutely positioned items position relative to the nearest positioned ancestor. Since outerbox1 has position: relative; set, innerbox1, is positioned at the top right corner.

Experiment 1: Remove position: relative from the #outerbox1 rule and refresh the page. Notice that the red box goes to the top left corner of the browser window. This is because there is no positioned ancestor anymore, so the box is positioned relative to the window.

Experiment 2: Put position: relative back into the #outerbox1 rule and then change top: 0; and left: 0; in the #innerbox1 rule to right: 0; and bottom: 0; and you should see the red box move to the bottom right corner of the yellow box.

Experiment 3: Try setting different pixel amounts for the top or bottom properties, and left or right properties to see what you get. Try both positive and negative numbers for each. Notice you can not set BOTH the top AND bottom properties, or BOTH the left AND right properties. You can position the red box anywhere inside or outside the yellow box, using this coordinate system.

Experiment 4: Set the top property to -200px, and the left property to 300px on the #innerbox1 rule and notice that the red box is now covering the text in the paragrahs above the yellow box. This is because absolutely positioned elements are taken completely out of the flow of the document and are ignored by other elements.

Example 2

Experiment 1: Add the top property and set it to -200px to the #innerbox2 rule. You will see that it moves the box, but it leaves a space where the box would normally display. For this reason, you don't usually use top/bottom or left/right properties with position: relative;. Usually position: relative; is just used to set a reference point for absolutely positioned elements, as shown in example 1 above.

Experiment 2: Change the positioning for #innerbox2 to position: absolute; to see what happens to the paragraph inside #outerbox2. You will see that when the positioning is switched, the paragraph now moves, because once #innerbox2 is set to absolute positioning, it is taken out of the flow of the document entirely.

You can move elements that have position: relative; set on them with the top/bottom, left/right properties, but they leave a space where the element would normally be displayed. Notice this paragraph stays where it is, if you follow the instructions in the experiment above.

Example 3

Experiment 1: Change the top property for #innerbox3 from 200px to -50px and see what happens. Because the parent element, #outerbox3 has overflow: hidden; set, anything overflowing that element (spilling outside it) is hidden, so only half the #innerbox3 is showing. This is useful for when you want to think of #outerbox3 as a stage that elements can enter and leave.

Experiment 2: Try setting left: 50%; and top: 50% on #innerbox3. You will see that puts the left side of the box half way across the #outerbox3 and the top of the box halfway down the #outerbox3.

Experiment 3 Try adding transform: translate(-50%, -50%); to #innerbox3. This is a way of centering elements with absolute positioning. There are other ways of centering elements using CSS flexbox and Grid, but this is sometimes useful.

Example 4

Experiment 1: In this example there are three absolutely positioned elements. Notice the third element is on top. That is because it loads last. Try adding z-index: 1; to #innerbox4a, and you should see it move to the top.

Experiment 2: Remove z-index: 1; from #innerbox4a and add z-index: -1; to #innerbox4c and you should see it move to the back. In fact, it looks like it disappeared entirely, but now it is behind #outerbox4.

Experment 3: Set z-index: 1; on #innerbox4a and set z-index: 2; on #innerbox4b and remove the z-index property (or set it to 0) on #innerbox4c. That will put the second box on top, then the first box, and then the third box on the bottom.

The property, z-index, will only affect elements with some sort of explicit positioning set, such as positoning: relative; or positioning: absolute. Also, z-index is 0 by default and can be used with positive or negative integers to create a lot of layers within a stacking context.
